Source Code Management

  • What was changed by whom, why and when?

Git - /ɡɪt/

  • Distributed Version Control System
  • Multiple, non-centralized repositories with their own full history
  • Easy collaboration between repositories

Let's get started - install Git!

  • Linux:
    • yum install git-core
    • apt-get install git-core
  • Mac: http://bit.ly/T3DD12-GitMac
  • Windows: http://bit.ly/T3DD12-GitWin

You should see be able to do this:

git --version git version 1.7.5.4 git --help

Let's get started - configure.

Minimal configuration - who are you?

git config --global user.name "Your Name" git config --global user.email "you@example.com"

Make yourself a home

git config --global core.editor vim git config --global color.diff auto git config --global color.status auto git config --global color.branch auto git config --global pager.status yes

Let's get started

Creating a local repository:

mkdir t3dd12; cd t3dd12 vim hello.php <?php echo 'Hello, World!!'; <esc>:wq git init Initialized empty Git repository in /home/you/t3dd12/.git git add hello.php git commit -m "First commit"

Repository status

git status # On branch master nothing to commit (working directory clean) vim hello.php # On branch master # Changed but not updated: # (use "git add <file>..." to update what will be committed) # (use "git checkout -- <file>..." to discard changes in working directory) # # modified: hello.php # no changes added to commit (use "git add" and/or "git commit -a") git add hello.php git commit -m "Hello world interactive."

Changes != files

vim hello.php git add hello.php vim hello.php git status # On branch master # Changes to be committed: # (use "git reset HEAD <file>..." to unstage) # # modified: hello.php # # Changed but not updated: # (use "git add <file>..." to update what will be committed) # (use "git checkout -- <file>..." to discard changes in working directory) # # modified: hello.php #

Undo...

Remove change before they've been staged

vim hello.php git checkout hello.php

Removing changes from the index:

vim hello.php git add hello.php git reset HEAD head.php vim hello.php

Partially adding things:

git add -p hello.php git status

History

What have we done so far?

git log git log --pretty=oneline

Adding details:

git log --pretty=format:"%h %ad | %s%d [%an]" --graph \ --date=short

Let's remember this one:

git config --global alias.hist '!git log \ --pretty=format:"%h %ad | %s%d [%an]" \ --graph --date=short' git hist

Tagging

Create tags:

git tag interactive git checkout 4193941 git tag first

List tags:

git tag git hist master --all

Remove tags:

git tag -d <tag>

Branches

Creating branches:

git branch greet git checkout greet # shortcut: git checkout -b greet

List branches:

git branch -av

Remove branches:

git branch -D <branch>

Merging branches

git checkout greet git merge master git hist --all * 3b58649 2012-04-09 | Merge branch 'master' into greet (HEAD, greet) |\ | * 1caec74 2012-04-09 | Added README (master) | * c37ca62 2012-04-09 | Extend comment * | b059311 2012-04-09 | Using the Greeter class * | b309cbf 2012-04-09 | Added greeter class |/ * ac319bf 2012-04-09 | Use a default value (interactive) * 049d36b 2012-04-09 | Added a comment * b866825 2012-04-07 | Interactive hello world * 4193941 2012-04-07 | First commit (first)

Resolving conflicts

Create the conflict:

git checkout master vim lib/Greeter.php git add lib/Greeter.php git commit -m "Prepare greeter" git checkout greet git merge master

Resolve it:

vim lib/Greeter.php git add lib/Greeter.php git commit -m "Merged master - kept Greeter"

Rebase for a new history

Rebase your current branch to be based on a "new" master: git rebase master Reorder, squash and drop commits with interactive commits: git rebase -i HEAD~3

Cherry-Picking

Picking "some" commits from one branch within others: git cherry-pick b059311

Remote repositories

Connecting with other repositories: git remote add origin git@github.com/user/repo git push origin git fetch origin git pull origin Using remote repositories right from the beginning git clone git@github.com/user/repo

Stash

A safe place to keep your uncommitted ideas: git stash From where your can retrieve everything back once you need it: git stash pop git stash apply

Tips

See wiki.typo3.org/Tips_and_Tricks_(Git).

  • Granular commits:git add -p
  • Highlight broken whitespaces before it's too late:
    git config color.diff.whitespace "red reverse"
  • Maintain global ignore lists:
    git config core.excludesfile ~/.git/.gitignore
  • Use commit message templates:
    git config commit.template <file>
